## 商店标题

Olyq

## 软件包摘要 / Manifest Description

浏览器里的开源多模型 AI 工作台，把网页上下文、Paint、搜索和远程 MCP 放进侧边栏。

## 建议类别

生产力 / Productivity

如果 Chrome Web Store 现有条目必须暂时保留原类别，可以继续使用“开发者工具”，但公开说明仍按浏览器里的多模型 AI 工作台口径填写。

## 详细说明

Olyq 是浏览器里的开源多模型 AI 工作台。它把当前网页、多模型对话、助手和话题、Paint、搜索、远程 MCP、截图 / OCR 和本地备份放进同一个侧边栏。

你可以用 Olyq 在网页旁边完成这些工作：

- 基于当前页面、选区、页面元素和截图向模型提问
- 用同一个问题对比多个已配置模型的回答
- 截图、标注、遮挡敏感区域，并把 OCR 结果继续用于当前话题
- 在 Paint 工作区使用你配置的图片模型，保留提示词、输入图、参数和历史结果
- 按需接入联网搜索、模型内置搜索和 remote MCP 工具
- 保留助手、话题、消息、附件、记忆和备份，方便继续同一段工作

Olyq 不内置托管模型账号，不提供免费模型额度，也不替你选择模型供应商。你需要在设置里添加自己的模型服务和 API Key，例如 OpenAI、Anthropic、Gemini、OpenRouter、Ollama 或其它兼容服务。实际可用模型取决于你的账号、服务区域、API Key 和模型权限。

数据与权限说明：

- 设置、助手、话题、消息、附件、记忆和备份默认保存在浏览器本地
- 只有当你配置并使用模型、搜索、remote MCP、WebDAV 或 S3-compatible 备份时，相关数据才会发往对应服务
- 当前页面内容、选区、页面元素和截图只在你使用网页上下文、页面工具、截图或 OCR 等功能时参与处理
- 扩展声明普通网页访问权限，用于页面上下文、页面工具、截图、元素引用和链接预览等浏览器扩展能力；这不代表 Olyq 会默认把每个网页发送给模型

Olyq 是独立开源项目。源码、隐私说明、安全说明和发布包校验信息见项目主页与 GitHub Releases。

## 隐私与权限说明

商店 privacy practices 应与 `PRIVACY.md` 保持一致：

- Olyq 不托管模型账号，不销售用户数据，不运行广告追踪，也不把页面内容发送到 Olyq 自有分析服务。
- 模型请求、图片生成、OCR 和记忆向量会发送给用户配置的模型服务。
- 联网搜索、模型内置搜索、remote MCP、OAuth、WebDAV 和 S3-compatible 备份只在用户配置并调用对应能力时访问外部服务。
- API Key、OAuth token、provider header、MCP 配置、WebDAV 配置和 S3-compatible 备份配置保存在浏览器 profile 中。
- 安装期普通网页访问权限用于页面上下文、页面工具、截图、元素引用和链接预览等能力；页面内容只在相关功能需要时参与处理。

## 审核测试说明

Olyq 是浏览器里的开源多模型 AI 工作台，不内置托管模型账号。

测试步骤：

1. 安装扩展。
2. 打开普通 `http` / `https` 网页。
3. 从工具栏打开 Olyq side panel。
4. 在 Settings 里添加受支持模型 provider 的测试 API Key。
5. 创建或选择助手与话题，围绕当前页面提问。
6. 测试选中文本、截图标注、OCR、多模型对比、Paint 或 remote MCP 等功能。

数据默认保存在浏览器本地。只有测试者配置模型、搜索、remote MCP 或备份 provider 并主动调用对应功能时，才会发生外部请求。
